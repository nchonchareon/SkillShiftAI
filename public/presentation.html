❌ แบบเดิม — Raw Text

"ทำตำแหน่งธุรการบัญชี คีย์เอกสารและคอยประสานงานทั่วไป ใช้คอมพิวเตอร์เป็นหลัก"

ผลลัพธ์: AI วิเคราะห์ได้กว้างเกินไป ไม่สามารถจัดทำแผนพัฒนาเฉพาะด้านได้

✅ แบบใหม่ — Guided Structured Input

ตำแหน่ง: ธุรการบัญชี
ภารกิจ: คีย์ข้อมูล Invoice ลง ERP, สรุปยอดรายจ่ายลง Excel
เครื่องมือ: MS Excel (VLOOKUP), ERP ภายใน

ผลลัพธ์: แตกภารกิจ ประเมินคะแนนความเสี่ยง พร้อมแนะนำคอร์สเรียนได้ทันที

สถาปัตยกรรมระบบ
ระบบ工作的ทั้งหมดทำงานอย่างไร?
แพลตฟอร์มแบ่งการทำงานออกเป็น 4 เลเยอร์หลัก ทำงานแบบ Event-Driven

📥 1. Data Ingestion

รับข้อมูล Job Description ผ่าน Guided Form หรือ Raw Text → แปลงเป็น JSON Structure

🤖 2. AI & Embedding Core

Ollama + Qwen3 วิเคราะห์แตกย่อยภารกิจ (Task Extraction) + แปลงทักษะเป็น Vector (Skill Embedding)

🗄️ 3. Vector Database (PostgreSQL + pgvector)

เปรียบเทียบหาความห่างพิกัดทักษะด้วย Cosine Similarity → จับคู่เส้นทางการพัฒนา

📊 4. Dashboard & Reporting

แสดงผลแผนภาพความเสี่ยง, Skill Gap, และแนะนำคอร์สเรียนผ่าน Dashboard แบบ Interactive

🔗 ความสัมพันธ์ระหว่างตาราง

-- Vector Search Query
SELECT ms.skill_name, se.embedding
  FROM skill_embeddings se
  JOIN master_skills ms ON ms.id = se.skill_id
  ORDER BY se.embedding <=> '[vector]'
  LIMIT 5;

-- Skill Gap Analysis
SELECT js.skill_id,
  MAX(js.importance_score) -
  COALESCE(MAX(us.proficiency), 0) AS gap
  FROM job_requirements js
  LEFT JOIN user_skills us ON ...
  GROUP BY js.skill_id
  ORDER BY gap DESC;
OpenCode Prompting Strategies
5 เทคนิคการเขียน Prompt ให้ OpenCode ทำงานได้อย่างเต็มประสิทธิภาพ
เปลี่ยนจากการ "ขอโค้ดให้ดู" เป็น "สั่งให้ลงมือทำจริง"

💡 1. สั่งให้ลงมือทำจริง

แทนที่จะขอโค้ดมา copy ให้สั่งตรง ๆ:
"ช่วยสร้างฟังก์ชันคำนวณภาษีใน src/utils/tax.ts แล้วเขียน Unit test ใน tax.test.ts"

💡 2. Plan แล้วค่อย Build

กด Tab ไป Plan Mode วิเคราะห์ก่อน แล้วกด Tab กลับ Build Mode ลุย:
"ลุยตามแผนที่วางไว้ได้เลย เสร็จแล้วรัน npm run test"

💡 3. ใช้ agents.md เป็นคัมภีร์

สร้างไฟล์ agents.md ที่ Root เพื่อกำหนด Tech Stack, Coding Style, และกฎเหล็กของโปรเจกต์ — AI จะอ่านทุกครั้งก่อนเริ่มงาน

💡 4. ระบุ Definition of Done

บอกให้ชัดว่าจะเช็คผลลัพธ์อย่างไร:
"แก้บั๊กแล้วรัน npm run lint และ npm test เพื่อเช็คก่อนส่งงาน"

💡 5. ใช้ Subagent สำหรับโปรเจกต์ใหญ่

ใช้ @explore นำหน้า Prompt เพื่อสแกนโค้ดเบื้องต้น:
@explore ช่วยหาว่าระบบ Authentication เก็บ JWT Token ไว้ที่ไฟล์ไหน
